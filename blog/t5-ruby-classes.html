Ruby Classes

And the Dawn of the Dinosaurs

2014/11/09

Classes - The basic template from which objects are created.


Within Ruby, almost everything you deal with, is considered an object. Classes are no exception to this rule. One could use a couple of methods to create simple Ruby program, but creating a class is beneficial to the structural organization of a code once it starts getting more complex.


Here, I have created the class `Dinosaurs`, whose sole purpose is to create individual dinosaurs with ease. After creating a new uninitialized instance of a class, the `initialize` method in the `Dinosaurs` class, is passed the parameters (`genus`, `tl`, `diet`, and `mouth`) as local variables.


These local variables are equated to instance variables (variables beginning with `@` symbols), so that they can be passed along to all methods within the class. These instance variables allow the `info` method to use the parameters given.


The method prints them back to the user in a string, giving us a description about the dinosaurs that were created.
